# 밥동무 기능 명세

## 서비스 개요

어르신이 호스팅을 열고 같이 밥 먹을 봉사자를 구하는 서비스.
보호자가 어르신을 등록하고 호스팅을 관리하며, 봉사자가 호스팅에 신청해 매칭된다.

---

## 역할 (user_role)

| 역할 | 설명 |
|------|------|
| `volunteer` | 봉사자. 호스팅 조회 및 신청, 체크인, 후기 작성 가능 |
| `guardian` | 보호자. 어르신 등록 및 호스팅 개설/관리 가능 |
| `admin` | 관리자. 신원검증 승인/반려, 통계 조회 가능 |

---

## 기능 목록

### 1. 회원 (인증)

| 기능 | 설명 |
|------|------|
| 회원가입 | 이름, 이메일, 비밀번호, 전화번호, 주소, 역할 입력. 봉사자는 서류 업로드 필수 |
| 로그인 / 로그아웃 | 이메일 + 비밀번호, JWT 발급 |
| JWT 발급 및 검증 | Access Token 기반 인증 |
| 신원검증 | 휴대폰 인증 (CoolSMS/Solapi 또는 알리고 활용) |
| 서류 업로드 | 봉사자 가입 시 필수. 파일 1개, `documents` 테이블에 저장 |

**비즈니스 로직**
- 서류 업로드 후 관리자가 승인하면 `users.cert_flag = approved`로 변경
- 반려 시 `cert_flag = rejected`로 변경, 기존 서류 삭제 후 재업로드 필요
- `cert_flag = approved` 상태인 봉사자만 호스팅 신청 가능

---

### 2. 마이페이지

| 기능 | 설명 |
|------|------|
| 내 정보 조회 / 수정 | 이름, 전화번호, 주소 등 수정 가능. 주소는 주소 API 연동 예정 |
| 서류 관리 | 업로드한 서류 조회/수정/삭제 |

---

### 3. 어르신 (보호자 전용)

| 기능 | 설명 |
|------|------|
| 어르신 등록 / 수정 / 삭제 | 보호자만 가능. 이름, 성별, 나이, 주소, 특이사항, 수용인원 입력 |
| 어르신 활성 상태 관리 | `active_flag`로 활성/비활성 관리 |
| Gemini AI 소개글 생성 | 후기 3건 이상 누적 시 후기 기반으로 자동 생성. `seniors.ai_summary`에 저장 |

---

### 4. 호스팅 (보호자 전용)

| 기능 | 설명 |
|------|------|
| 호스팅 등록 / 수정 / 삭제 | 보호자만 가능. 메뉴, 일시, 모집인원 입력 |
| 호스팅 조회 | 봉사자는 목록 조회 가능. 단, 상세 주소는 봉사자에게 비공개 |
| 모집 인원 기본값 | `seniors.max_people` 값을 기본값으로 사용, 호스팅별 수정 가능 |

**호스팅 상태 (`hosting_status`)**

| 상태 | 설명 |
|------|------|
| `신청가능` | 모집 중 |
| `모집완료` | 정원 마감 |
| `신청불가` | 취소 또는 비활성 |

**비즈니스 로직**
- 호스팅 수정 시 → 신청한 봉사자 전원에게 SMS 발송 (`alarm_type: update`)
- 신청 인원이 `max_people` 도달 시 → `hosting_status` 자동으로 `모집완료`로 변경

---

### 5. 매칭

| 기능 | 설명 |
|------|------|
| 매칭 신청 | 봉사자가 호스팅에 신청. `cert_flag = approved` 필수 |
| 매칭 취소 | 봉사자가 자발적으로 취소 가능. `is_apply = false` |
| 체크인 | 봉사자가 방문 당일 직접 체크인. `check_in = true` |

**비즈니스 로직**
- 신청 시 → 보호자에게 SMS 발송 (`alarm_type: match`)
- 체크인 시 → 보호자에게 SMS 발송 (`alarm_type: checkin`)
- 취소 시 → `hosting_status`가 `모집완료`였다면 `신청가능`으로 되돌림

---

### 6. 후기

| 기능 | 설명 |
|------|------|
| 후기 작성 / 수정 / 조회 | 매칭 완료(체크인)된 봉사자만 작성 가능 |
| 후기 이미지 첨부 | 최대 5개, `review_img` 테이블에 저장 |

---

### 7. 관리자

| 기능 | 설명 |
|------|------|
| 신원검증 승인 / 반려 | 봉사자 서류 확인 후 승인 → `users.cert_flag` 변경 |
| 대시보드 통계 조회 | 유효 등록자별 방문 횟수 등 유의미한 데이터 파악 |

---

## SMS 발송 트리거 정리

| 트리거 | 발송 시점 | 수신자 | alarm_type |
|--------|-----------|--------|------------|
| 매칭 신청 | 봉사자가 호스팅 신청 시 | 보호자 | `match` |
| 체크인 | 봉사자가 방문 체크인 시 | 보호자 | `checkin` |
| 호스팅 수정 | 호스팅 정보 변경 시 | 신청한 봉사자 전원 | `update` |

> SMS 서비스: Solapi(CoolSMS) 또는 알리고 사용 예정  
> 수신자가 여러 명인 경우 수신자별로 `sms_logs` row 생성

---

## 외부 서비스 연동

| 서비스 | 용도 |
|--------|------|
| Solapi / 알리고 | SMS 발송 |
| Gemini AI | 어르신 소개글 자동 생성 (후기 3건 이상 시) |
| 주소 API | 마이페이지 주소 입력 연동 |
